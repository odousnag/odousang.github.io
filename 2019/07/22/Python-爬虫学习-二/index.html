3.Request

class urllib. request. Request(ur1, data=None, headers={}, origin_req_host=None,
unverifiable=False, method=None)

第一个 url用于请求URL 这是必传参数
第二个 data ，必须传 bytes （字节流）类型的。 如果它是字典，可以先用 urllib.parse 模块里的 urlencode（）编码。
第三个 headers 是请求头，在构造请求时通过 headers 参数直 接构造，也可以通过调用请求实例的 add_header（）方法添加。
最常用的用法就是通过修改 User-Agent 来伪装浏览器，默认的 User-Agent 是 一－ Python-urllib ，我们可以通过修改它来伪装浏览器。 比如要伪装火狐浏览器，你可以把它设置为 ：
Mozilla/5.o (X11; U; Linux i686) Gecko/20071127 Firefox/2.0.0.11
第四个 origin_req_host 是请求方的host名称或者IP地址
第五个 unverifiable 默认是false 意思是用户没有足够的去耐心啊来接收这个请求的结果
第六个 method 用来表示请求的方法 GET POST PUT

4.高级用法

先学习一下 urllib.request 模块里面的 BaseHandler 类
里面会有各种 Handler 子类继承
HTTPDefaultErrorHandler ：用于处理http响应错误 抛出异常
HTTPRedirectHandler：用于处理重定向
HTTPCookieProcessor：用于处理Cookies
ProxyHandler：用于设置代理，默认代理为空
HTTPPasswordMgr：用于管理密码，维护用户名和密码的表
HTTPBasicAuthHandler：用于管理认证，一个链接打开时需要认证，那么可以来解决认证问题

另一个重要的类为 Opener 可以使用open()方法，返回类型和urlopen()如出一辙

我们遇到这个问题，借助 HTTPBasicAuthHandler 就可以完成

这里首先实例化 HTTPBasicAuthHandler 对象，其参数是 HTTPPasswordMgrWithDefaultRealm 对象， 它利用 add_password（）添加进去用户名和密码，这样就建立了一个处理验证的 Handlero
接下来，利用这个 Handler 并使用 build_opener（）方法构建一个 Opener，这个 Opener 在发送请求 时就相当于已经验证成功了。
接下来，利用 Opener 的 open（）方法打开链接，就可以完成验证了。

2.处理异常

1.URLError

来自 urllib 库的error模块，由request模块产生的异常都可以通过这个类来处理

这时候我们捕获了这个异常 运行结果就会出现 ：Not Found

2.HTTPError

它是URLError 的子类，专门来处理HTTP请求错误。它有三个属性
code：返回HTTP状态码 例如404表示网页不存在 500表示服务器内部错误
reason：返回错误的原因
headers：返回请求头

这样做可以先捕获HTTPError异常，获取异常原因，错误状态码，headers信息
不是HTTPError异常，就会捕获URLError异常，输出错误原因
最后用else来处理正常逻辑

4.分析 Robots 协议

利用 urllib 的 robotparser 模块，可以实现网站 Robots 协议的分析。

1.Robots 协议

这些协议也称为爬虫协议，机器人协议。它通常是一个叫作 robot.txt 的文本文件，一般放在网站的根目录下。
当搜素爬虫访问一个站点时，它会检查这个站点根目录下是否存在 robot.txt 文件。
如果存在，爬虫就会根据其中定义的爬取范围来爬取，如果不存在，爬虫便会访问所有可直接访问的页面

3.robotparser

我们就可以用 robotparser 模块来解析 robots.txt 。
该模块提供了一个 类 RobotFileParser，它可以根据某网站的 robots.txt 文件来判断一个爬取爬虫是否有权限来爬取这个网页。

这个类常用的几个方法。
set_url()：用来设置 robots.txt 文件的链接。
read()：读取 robots.txt 文件并进行分析。 注意，这个方法执行一个读取和分析操作，如果不调用这个方法， 接下来的判断都会为 False，所以一定记得调用这个方法。 这个方法不会返回任何内容，但是执行了读取操作。
parse()：用来解析robots.txt文件，传人的参数是 robots.txt某些行的内容，它会按照 robots.txt 的语法规则来分析这些内容。
can_fetch()：该方法传人两个参数， 第一个是 User-agent，第二个是要抓取的 URL。 返回的 内容是该搜索引擎是否可以抓取这个 URL，返回结果是 True 或 Falsea
mtime()：返回的是上次抓取和分析 robots.txt 的时间，这对于长时间分析和抓取的搜索爬虫是 很有必要的，你可能需要定期检查来抓取最新的 robots.txt。
modified()：它同样对长时间分析和抓取的搜索爬虫很有帮助，将当前时间设置为上次抓取和分析 robot.txt的时间

首先创建 RobotFileParser 对象， 然后通过set_url()方法设置了 robots.txt 的链接
同样可以使用parse()方法执行读取和分析

3.超时设置

在本机网络状况不好或者服务器网络响应太慢甚至无响应时，甚至到最后收不到响应而报错。
为了防止服务器不能及时响应，设置一个超时时间，即使超过了这个时间还没有得到响应，那就报错。
这里需要timeout参数。这个时间的计算是发出请求到服务器返回响应的时间。

通过这样的方式，我们可以将超时时间设置为 l 秒，如果 l 秒内没有响应，那就抛汁l异常。
请求分为两个段落，连接（connect）和读取（read）。
上面设置的timeout将用作连接和读取这二者的timeout总和。
如果想永久等待，可以直接将timeout设置为None，或者不设置直接留空，因为默认是None。
